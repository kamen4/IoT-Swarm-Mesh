Class FloodingPacketRouter

Implements a flooding broadcast routing strategy.

When asked to route a packet the router queries the supplied INetworkTopology for all devices visible to the sender and enqueues a cloned copy addressed to the nearest visible neighbors in a deterministic top-k window. Flooding uses visibility - not established connections - because it is inherently a broadcast mechanism that needs no prior topology knowledge.

This class is no longer a singleton. The active router instance is stored on Router and is injected into every routing call automatically by Recieve(Packet). To change the routing strategy at runtime replace Router with a different IPacketRouter implementation.

Remarks

Broadcasts packet to every device that is within VisibilityDistance of sender but limits fan-out to Engine.Routers.RoutingNeighborPolicy.DefaultMaxVisibleNeighbors nearest visible neighbors. Each visible neighbour receives its own shallow clone so that per-hop fields (NextHop, ArrivalTick) can be set independently.

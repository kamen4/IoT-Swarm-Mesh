Class BenchmarkRunner

Namespace
Engine.Benchmark
Assembly
Engine.dll

Executes a BenchmarkConfig headlessly - without any UI - and produces a BenchmarkSession containing one BenchmarkResult per configured router.

Threading: supports both synchronous and asynchronous execution. The async entry point periodically yields to keep UI render loops responsive on single-threaded environments such as Blazor WebAssembly. Each router run uses an isolated engine/statistics context so independent runs can execute in parallel when requested.

Progress reporting: the optional onProgress callback in Run(BenchmarkConfig, IReadOnlyDictionary<string, IPacketRouter>, Action<BenchmarkRunProgress>?, INetworkBuilder?, SwarmProtocolVector?, int) receives detailed BenchmarkRunProgress snapshots while each router is running, including per-router tick progress and overall progress.

Run(BenchmarkConfig, IReadOnlyDictionary<string, IPacketRouter>, Action<BenchmarkRunProgress>?, INetworkBuilder?, SwarmProtocolVector?, int)

Runs the benchmark described by config synchronously. Call this inside Task.Run() to avoid blocking the UI thread.

public static BenchmarkSession Run(BenchmarkConfig config, IReadOnlyDictionary<string, IPacketRouter> availableRouters, Action<BenchmarkRunProgress>? onProgress = null, INetworkBuilder? networkBuilder = null, SwarmProtocolVector? swarmVector = null, int maxDegreeOfParallelism = 1)

Parameters

config BenchmarkConfig

The benchmark configuration to execute.

availableRouters IReadOnlyDictionary<string, IPacketRouter>

All known IPacketRouter implementations, keyed by Name. Routers listed in RouterNames that are not found here are silently skipped.

onProgress Action<BenchmarkRunProgress>

Optional callback invoked during execution with detailed BenchmarkRunProgress snapshots. Called on the thread-pool thread, not the Blazor synchronisation context.

networkBuilder INetworkBuilder

Optional topology builder override for this benchmark run. When null, the current Instance builder is used as a template.

swarmVector SwarmProtocolVector

Optional swarm-vector override for this benchmark run. When null, the current Instance vector is used as a template.

maxDegreeOfParallelism int

Maximum number of router runs executed in parallel. Use 1 for fully sequential execution.

Returns

BenchmarkSession

A BenchmarkSession with results for every router that was found and executed.

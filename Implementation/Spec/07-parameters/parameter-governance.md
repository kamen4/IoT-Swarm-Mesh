# Parameter Governance Process

## Change lifecycle

1. Propose parameter change with rationale.
2. Map expected impact on convergence and stability.
3. Apply staged rollout on controlled subset.
4. Validate against delivery and oscillation indicators.
5. Promote, revise, or rollback.

## Required records

- Parameter set identifier.
- Effective deployment scope.
- Observed before/after metrics.
- Decision outcome and owner.

## Guardrails

- No silent parameter drift.
- No emergency change without explicit incident record.

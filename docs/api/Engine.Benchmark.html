AddDeviceBenchmarkEvent

Adds a new device to the engine at the scheduled tick, simulating a node joining the network mid-simulation.

BenchmarkConfig

Fully serialisable description of a benchmark scenario.

A BenchmarkConfig defines everything needed to reproduce a headless simulation run deterministically:

  • Simulation engine settings (visibility, TTL, packet travel time).
  • The initial network layout as a list of DeviceBenchmarkDto.
  • A timeline of BenchmarkEventEntry items that modify the network at specific ticks (toggle, add, remove device).
  • The total number of ticks to simulate.
  • The list of router names to test; BenchmarkRunner will run one full simulation per router and collect results for comparison.

Instances are stored as JSON together with their BenchmarkSession results so a test can be replayed or shared.

BenchmarkEvent

Base type for all events that can be scheduled in a BenchmarkConfig.

BenchmarkEventEntry

Associates a BenchmarkEvent with the engine tick at which it should be fired during a benchmark run.

BenchmarkResult

The collected statistics from one headless simulation run for a single router protocol.

BenchmarkRunner

Executes a BenchmarkConfig headlessly - without any UI - and produces a BenchmarkSession containing one BenchmarkResult per configured router.

Threading: designed to be called from Task.Run so that the Blazor UI thread stays responsive during the (potentially long) run. All engine interaction happens through the singleton Instance which is fully reset between runs.

Progress reporting: the optional onProgress callback in Run(BenchmarkConfig, IReadOnlyDictionary<string, IPacketRouter>, Action<int, int>?) is invoked after each completed router run with (routerIndex, totalRouters) so callers can display a spinner.

BenchmarkSession

The complete, serialisable record of one benchmark execution. Stored as JSON so users can save, share, and reload results.

DeviceBenchmarkDto

Serialisable snapshot of a single device's configuration used inside BenchmarkConfig. Converted to a live Device by BenchmarkRunner when it builds the engine state. Declared as a record so Blazor event handlers can use with expressions to create modified copies without mutating shared state.

RemoveDeviceBenchmarkEvent

Removes the named device from the engine at the scheduled tick, simulating a node failure or graceful leave.

ToggleBenchmarkEvent

Hub sends a toggle (on/off) ControlPacket to the named emitter device at the scheduled tick.

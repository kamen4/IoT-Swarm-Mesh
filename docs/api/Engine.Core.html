PacketLimitExceededException

Thrown by RegisterPacket(Packet) when the number of in-flight packets in the priority queue reaches or exceeds MaxActivePackets.

This exception signals that the simulation has entered an unstable state (packet storm / routing loop) where more packets are being generated than the network can drain. The host should catch this exception, stop the tick loop, and surface the error to the user.

SimulationEngine

The central simulation engine type that drives the entire simulation. It maintains the device registry, owns the tick-based time model, and manages the priority queue of in-flight packets.

Every call to Tick() advances the simulation by one step: the tick counter is incremented, wall-clock delta time is measured, all subscribers to TickEvent are notified, and any packets whose ArrivalTick has been reached are dequeued and dispatched to their NextHop device.

Routing strategy and network-formation strategy are fully replaceable at runtime via Router and NetworkBuilder. The read-only topology view is exposed through Topology.

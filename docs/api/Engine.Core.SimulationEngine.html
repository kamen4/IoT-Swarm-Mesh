Class SimulationEngine

Namespace
Engine.Core
Assembly
Engine.dll

The central singleton that drives the entire simulation. It maintains the device registry, owns the tick-based time model, and manages the priority queue of in-flight packets.

Every call to Tick() advances the simulation by one step: the tick counter is incremented, wall-clock delta time is measured, all subscribers to TickEvent are notified, and any packets whose ArrivalTick has been reached are dequeued and dispatched to their NextHop device.

Routing strategy and network-formation strategy are fully replaceable at runtime via Router and NetworkBuilder. The read-only topology view is exposed through Topology.

MaxActivePackets

Gets or sets the maximum number of packets that may be simultaneously in-flight in the simulation.

When set to 0 (the default) the limit is computed automatically as Devices.Count × AUTO_PACKET_LIMIT_PER_DEVICE each time a packet is enqueued, so it scales with the network size.

When the in-flight count reaches or exceeds the effective limit, RegisterPacket(Packet) throws PacketLimitExceededException and the tick loop must be stopped by the caller. Set to a positive value to override the automatic limit. 

public int MaxActivePackets { get; set; }

Triggers the active NetworkBuilder to recompute the logical connection graph for the current device set.

Called automatically on every device-registry change (RegisterDevice(Device), RemoveDevice(Guid)). Can also be called manually — for example after moving a device or after replacing NetworkBuilder — to immediately reflect the new topology without waiting for the next device change.

Tick()

Advances the simulation by one tick.

  1. Increments TickCount.
  2. Measures wall-clock delta time since the last tick.
  3. Fires TickEvent so that subscribed devices can act.
  4. Dispatches all packets that are due to arrive this tick.
  5. Raises Ticked.

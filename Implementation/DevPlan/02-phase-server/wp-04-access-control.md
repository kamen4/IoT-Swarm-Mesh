# Server WP-04 Access Control

## Goal

Define and enforce role-based command and administration policy.

## Tasks

T1. Permission matrix definition
- T1.1 Enumerate command categories and admin actions.
- T1.2 Map permissions for User, DedicatedAdmin, Admin.
- T1.3 Define denied-action behavior and audit requirements.

T2. Enforcement points
- T2.1 Define enforcement at command intake.
- T2.2 Define enforcement at administrative endpoints.

T3. Governance controls
- T3.1 Define role assignment and revocation workflow.
- T3.2 Define role change audit events.

## Deliverables

- Role-permission matrix.
- Access enforcement specification.

## Acceptance criteria

- Every user-visible action has explicit permission policy and audit behavior.

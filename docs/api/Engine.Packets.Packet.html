Class Packet

Namespace
Engine.Packets
Assembly
Engine.dll

Represents a unit of data travelling through the simulation network. A packet knows its originator (From), its intended destination (To), and the NextHop device it will be delivered to on the next simulation tick. Travel time is expressed in engine ticks via TicksToTravel, and each hop decrements the TTL counter - a packet that reaches zero TTL is silently dropped.

The same class also carries a swarm protocol envelope:

  • Routing metadata: version, previous-hop MAC, advertised charge and decay hint.
  • Secure header fields: direction, message type, origin/destination MAC, message id and sequence.
  • End-to-end tag bytes used by HMAC-SHA256 truncation.
These fields are optional for legacy routing and are populated by default for newly created packets.

OriginId

Gets the identifier shared by this packet and all of its flood clones. On the original packet OriginId == Id. After Clone(), every copy keeps the same OriginId because MemberwiseClone copies the field as-is.

SimulationStatistics uses this to detect duplicate deliveries: when a second clone with the same OriginId reaches the destination it is counted as a duplicate rather than a new delivery.

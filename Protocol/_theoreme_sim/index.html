Theorem and simulation guide

What exactly is proven

The theorem checks are structural checks over the current charge field and parent pointers. They verify that, on the eligible subset, parent edges strictly increase charge, stay acyclic, and eventually reach the gateway.

  • A5: gateway charge is the unique strict maximum among eligible nodes.
  • A6: each eligible non-root node has an eligible neighbor with higher charge.
  • A7: parent is selected among neighbors with strictly higher charge.
  • Lemma 4.1/4.2/4.3: strict increase, acyclic parent graph, reachability to gateway.
  • PENDING means the eligible non-root set is still empty, so theorem checks are not informative yet.

Round pipeline (what one step does)

  1. DOWN phase: gateway emits one DOWN packet; forwarding starts as broad coverage and gradually becomes tree-dominant. Duplicate receptions are counted for this exact step.
  2. During DOWN forwarding, receiver updates neighbor estimate by max rule and updates local charge toward senderCharge - chargeDropPerHop.
  3. UP phase: each device tries to route toward gateway via parent or best higher-charge neighbor; successful hops increase traffic centrality.
  4. Neighbor propagation phase: edge deliveries update neighbor estimates with deliveryProbability.
  5. Spread smoothing phase: each node blends toward its best observed neighbor target with chargeSpreadFactor.
  6. Tree rebuild phase: parent selection uses strict charge increase, link-quality penalty, hysteresis, and link-stability bonus.
  7. DECAY phase (if configured interval hits): charges and stored estimates are multiplied by (1 - decayPercent), and epoch is incremented.
  8. Verification phase: A5/A6/A7 and lemmas are evaluated, oscillation metrics are updated.

Randomization and reproducibility

  • Topology generation and runtime packet losses are deterministic for a fixed seed.
  • Randomize All Params generates values in bounded ranges, then applies cross-parameter normalization for stability.
  • Normalization prevents pathological combinations, for example q_forward >= rootSourceCharge or unrealistic hop drop.
  • To reproduce a run exactly, keep the same seed and do not change parameters mid-run.

Why high-traffic links become stronger

Each edge tracks usage over rounds. Frequently used edges increase effectiveQuality, which increases parent preference and delivery reliability in stochastic phases. This naturally strengthens gateway-adjacent links and bottleneck bridges between clusters.

  • linkMemory controls how long historical usage remains influential.
  • linkLearningRate controls adaptation speed of effective link quality.
  • linkBonusMax limits how much parent scoring can favor a stable high-traffic edge.

How to interpret stability metrics

  • duplicates trend should generally decrease over time.
  • flapping nodes > 0 means repeated parent switching is still active.
  • gateway edge quality should be high in stable converged regimes.
  • strongest link often becomes either near-gateway trunk edge or a cluster bridge edge carrying concentrated traffic.
  • If structure collapses late in run, increase hysteresis, reduce spread factor, or use shorter decay interval.

Namespace Engine.Devices

Classes

Device

Abstract base class for all devices in the simulation network. A device has a unique identity and a position in 2D space. When it receives a packet it either forwards it (if it is not the intended recipient) or accepts and processes it.

Forwarding is delegated to RoutePacket(Packet, Device) so that the active IPacketRouter is always consulted. Devices never reference a concrete router directly.

EmitterDevice

A device that receives command packets from the hub and applies a boolean state to itself (e.g. turning an actuator on or off).

When ControlFrequencyTicks is greater than zero the hub automatically sends a ControlPacket to this device every ControlFrequencyTicks ticks, toggling its state from on to off and back. Set ControlFrequencyTicks to 0 to disable automatic control signals.

Flooding note: with FloodingPacketRouter a unicast ControlPacket is re-broadcast at every intermediate hop, creating an exponential number of clones (approximately neighbours^TTL in the worst case). The TTL is therefore capped at CONTROL_PACKET_TTL rather than the default packet TTL. When a unicast router is active this has no extra cost because the packet follows a single path.

GeneratorDevice

A device that periodically generates sensor-like data packets and routes them toward the hub.

Routing is delegated to RoutePacket(Packet, Device) so the active IPacketRouter strategy is always used.

HubDevice

Represents the central hub (gateway) device of the simulation network. The hub is the designated destination for packets generated by GeneratorDevice instances and the root of the network topology. It is registered as the singleton hub inside SimulationEngine when added to the simulation.

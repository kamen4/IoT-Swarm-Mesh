Class EmitterDevice

A device that receives command packets from the hub and applies a boolean state to itself (e.g. turning an actuator on or off).

When ControlFrequencyTicks is greater than zero the hub automatically sends a ControlPacket to this device every ControlFrequencyTicks ticks, toggling its state from on to off and back. Set ControlFrequencyTicks to 0 to disable automatic control signals.

Flooding note: with FloodingPacketRouter a unicast ControlPacket is re-broadcast at every intermediate hop, creating an exponential number of clones (approximately neighbours^TTL in the worst case). The TTL is therefore capped at CONTROL_PACKET_TTL rather than the default packet TTL. When a unicast router is active this has no extra cost because the packet follows a single path.
